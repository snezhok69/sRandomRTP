# sRandomRTP 3.1 — стабильность, админские debug-инструменты и более безопасный RTP

Админские debug-инструменты!

1. Следующие команды админ-мониторинга являются debug-инструментами и по умолчанию выключены в публичных установках через Settings/commands.yml:

/rtp tpsbar
/rtp rambar
/rtp msptbar
/rtp allbars

1.1. /rtp tpsbar — показывает или скрывает админский bossbar с текущим TPS сервера.
Пермишион: sRandomRTP.Command.TpsBar

1.2. /rtp rambar — показывает или скрывает админский bossbar с использованием RAM.
Пермишион: sRandomRTP.Command.RamBar

1.3. /rtp msptbar — показывает или скрывает админский bossbar с текущим MSPT.
Пермишион: sRandomRTP.Command.MsptBar

1.4. /rtp allbars — включает или выключает все админские bossbar сразу.
Пермишион: sRandomRTP.Command.AllBars

1.5. На публичных серверах эти команды не появляются в tab completion и /rtp help, пока они не включены в Settings/commands.yml или через /rtp settings.


Возможности админ-мониторинга!

1. Добавлены настраиваемые bossbar для TPS, RAM и MSPT для мониторинга сервера.

2. Для каждого bossbar можно отдельно настроить заголовок, цвет, стиль, формат обновления, пороги и состояние включения в admin-bars.yml.

3. Команды и tab completion учитывают пермишионы, состояние конфига и поддержку метрик на сервере.

4. Bossbar корректно очищаются при выходе игрока, перезагрузке плагина и выключении плагина.


Улучшения телепортации и поиска!

1. RTP был перестроен вокруг общих обработчиков для обычного /rtp, near, far, middle, biome, base, world, player и телепортации через порталы.

2. Добавлен менеджер жизненного цикла телепорта: он отслеживает активные запросы, отменяет заменённые/истёкшие/оффлайн-запросы и защищает от повторного финального телепорта.

3. Параллельный поиск координат стал адаптивным: плагин может ограничивать общий лимит поиска и снижать нагрузку при высоком давлении на чанки, TPS или MSPT.

4. Добавлена опция сначала искать уже сгенерированные чанки, и только потом разрешать генерацию новых.

5. Загрузка и предзагрузка чанков улучшены: безопасное планирование для Paper/Folia, срочные PaperLib-запросы, ограничение соседней предзагрузки и телеметрия нагрузки на чанки.

6. Проверка безопасной точки переписана вокруг загруженных чанков, снимков биомов/блоков, отдельных проверок Nether/End, опасных блоков, глубокой воды, листвы, наклонов и отмены поиска.

7. /rtp biome теперь использует отдельный biome.yml, принимает биомы через пробелы или запятые, нормализует регистр/дефисы/пробелы и раньше отсекает запрещённые биомы.

8. Плагин запоминает недавние точки телепорта, чтобы реже отправлять игрока обратно в ту же область.

9. Исправлены пермишионы кулдауна: sRandomRtp.Cooldown.N теперь стабильно работает и также задаёт длительность bossbar-отсчёта.


Обновление конфигурации и локализации!

1. Добавлены новые конфиги:

admin-bars.yml — настройки bossbar для TPS/RAM/MSPT. Команды по умолчанию выключены в Settings/commands.yml.
biome.yml — отдельные радиусы и настройки поиска для /rtp biome, включая per-world параметры и двухфазный поиск.

2. В конфигах появились управляемые config-version, синхронизация дефолтных ключей и миграции при запуске.

3. В config.yml добавлен metrics.rtp.slow-request-threshold-ms для диагностики долгих RTP-запросов.

4. В teleport.yml добавлены настройки parallel-search.max-global-inflight и prefer-generated-chunks.

5. Auto-reload теперь следит за основным config, Settings и lang, проверяет YAML перед применением и перезагружает язык только когда меняются языковые файлы.

6. Сообщение о неизвестной команде теперь можно менять через ключ локализации invalid-command.

7. Английская и русская локализации расширены сообщениями для admin bossbar, invalid command и обновлёнными подсказками команд. Публичный /rtp help скрывает debug-админ команды, пока их command-switch выключен.


Архитектура и диагностика!

1. Плагин теперь использует центральный PluginContext и сервисный доступ к общим системам.

2. Загрузка конфигов централизована через ConfigRegistry и неизменяемые снимки ConfigCache.

3. Глобальное runtime-состояние перенесено в RuntimeStateRegistry, чтобы reload, shutdown и очистка были безопаснее.

4. Проверка обновлений вынесена в ReleaseCheckService с кешем, защитой от дублей запросов и более заметными предупреждениями о старой версии.

5. DiagnosticsService показывает диагностику при запуске и reload, а TeleportMetrics умеет логировать долгие RTP-запросы и сводку по телепортам.

6. Код порталов разделён на отдельные классы для repository, state store, частиц, установки блоков, триггеров и cooldown.

7. SQL-хранилище порталов стало чище: асинхронное выполнение, индексы, версии схемы и миграции для формы портала.


Мелкие изменения и исправления!

1. LuckPerms больше не является обязательной зависимостью; опциональные интеграции обрабатываются мягче.

2. Новый bypass-пермишион: sRandomRTP.Command.bypass, при этом старый sRandomRTP.Cooldown.bypass всё ещё работает для совместимости.

3. Исправлено сообщение о неизвестной команде: теперь используется настраиваемый текст из локализации, а не стандартное сообщение сервера.

4. Улучшена очистка блоков порталов, частиц, admin bossbar, ожидающих телепортов и списаний экономики.

5. Исправлены случаи с повторными RTP-запросами, выходом игрока во время телепорта, истекшими запросами и возвратом денег после отмены.

6. Проверки пермишионов, сообщения команд, логика запрещённых миров и стартовые проверки телепортации централизованы, чтобы убрать дублирование поведения.

7. Метаданные плагина обновлены под релиз 3.1. Плагин собирается в Java 8 bytecode и сохраняет минимальную Bukkit API-метку 1.16, чтобы старые серверы 1.16 всё ещё могли его загрузить, а новые версии продолжали работать.

8. Целевая Java остаётся совместимой с Java 8, часть зависимостей обновлена, лишние build-зависимости удалены. Функции, завязанные на более новые API сервера, теперь используют fallback или остаются недоступными на старых ядрах.

9. Если сервер всё ещё работает на Java 8, sRandomRTP 3.1 должен загружаться. Для совсем старых сборок сервера запасными вариантами могут быть 3.0 или 2.9/2.9-FIX.

10. Добавлены автоматические тесты в tests/java: кулдауны, конфиги, миграции, admin bossbar, порталы, biome RTP, нагрузка чанков, safe-Y проверки и scheduler guards.

11. .gitignore обновлён, чтобы IDE-файлы, локальные файлы агентов, runtime-базы, логи и build-артефакты не попадали в репозиторий.


Дополнительная полировка стабильности!

1. Добавлены /rtp doctor, /rtp dump, /rtp stats и /rtp portal check, чтобы диагностика сервера и помощь пользователям были проще.

2. Chunk warming, частицы/триггеры порталов, admin bossbar и callbacks проверки версии теперь избегают небезопасного async-доступа к Bukkit API.

3. Счётчик RTP теперь сохраняется пачками, а не пишет Data/rtpCount.yml после каждого телепорта; RTP через cooldown bypass теперь тоже корректно считается.

4. При обновлении конфигов создаются backup-файлы и пишется Diagnostics/config-changes.txt, а успешная диагностика startup/reload сохраняется даже при выключенных обычных логах.

5. Добавлены PlaceholderAPI значения и публичные RTP/portal events для интеграций, а также тесты на версии конфигов, permissions в plugin.yml, scheduler safety и Java 8 bytecode.

6. Добавлен Settings/commands.yml и /rtp settings — кликабельное меню прямо в игре для включения/выключения RTP-подкоманд без удаления их permission-проверок.

7. Команды отладки/поддержки получили более понятные переключатели: /rtp doctor, /rtp dump, /rtp stats, /rtp portal check и admin bossbar-команды теперь переключаются из одной debug-секции.

8. /rtp portal check относится к debug/support-командам и теперь имеет отдельную permission-ноду: sRandomRTP.Command.Portal.Check.

9. Вывод новых debug/settings-команд теперь локализуется через lang/*.yml, включая /rtp doctor, /rtp dump, /rtp stats, /rtp settings, /rtp portal check и help-строки admin bossbar.

10. Admin bossbar-команды остаются в плагине, но в публичной сборке выключены по умолчанию через Settings/commands.yml; после включения /rtp settings можно отдельно переключать видимость /rtp tpsbar, /rtp rambar, /rtp msptbar и /rtp allbars.

11. /randomtp и /randomteleport перенесены из plugin.yml в config.yml → Command-Aliases, поэтому владелец сервера может включать, удалять или менять RTP-алиасы без правки metadata плагина.


Примечания

Обновление 3.1 в основном сфокусировано на стабильности, диагностике и внутренней чистке после большого релиза 3.0. Внутри сильно изменились телепорты, порталы, кулдауны и конфиги, поэтому если найдёте пропущенный edge case — пишите на Discord-сервер или создавайте GitHub issue.
